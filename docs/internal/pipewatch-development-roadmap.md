## How to use this roadmap

### With the Cursor orchestrator

1. Pick the next **leaf task** whose dependencies are `[x]` and status is `[ ]`.
2. Dispatch a sub-agent with: **Task ID**, **acceptance criteria (verbatim)**, **doc refs**, **design refs**, **read/write paths**.
3. Verifier runs `pnpm lint`, `pnpm typecheck`, `pnpm test:unit`, `pnpm test:integration` (when applicable), `pnpm build`.
4. Mark `[x]` on PASS; `[!]` on FAIL.

**Lane guidance:** Foundation phases (P0–P11) are mostly **Lane S** (direct to `staging`). Large UI page epics (P12–P16) can use **Lane P** (task branch → verify → merge).

### GitHub issue sync

**Status:** Complete — see [GitHub enrichment verification checklist](#github-enrichment-verification-checklist) (116/116 verified via MCP).

Each **Epic** (phase heading) → GitHub **Feature** issue (parent).  
Each **leaf** row → GitHub **Task** issue (child via `sub_issue_write`).

| Roadmap field | GitHub issue field |
|---|---|
| Task title | `[P12-02] App shell (sidebar, workspace switcher)` |
| Acceptance criteria bullets | Issue body § Acceptance criteria |
| Doc ref / Design ref | Issue body § References |
| Domain column | Label `domain:*` |
| Deps | Issue body § Dependencies |
| Tests | Issue body § Verification |

**Labels:** `domain:infrastructure` \| `domain:backend` \| `domain:frontend` \| `domain:operations` · `type:epic` \| `type:task` · `effort:XS|S|M|L|XL`

**Required on every issue:** Type, domain label, Priority, Effort, Assignee, Milestone (M1–M7), parent link for leaf tasks.

**Post-sync:** Optionally add `github: #NNN` inline on each leaf row for traceability.
