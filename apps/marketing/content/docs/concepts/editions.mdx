## One codebase

PipeWatch ships **Cloud** (managed at `cloud.pipewatch.app`) and **CE** (self-hosted Docker Compose) from the same monorepo. Edition behaviour is driven by `PIPEWATCH_EDITION` and `packages/config/edition.ts` flags — not duplicated implementations.

## Cloud-only features

- Stripe billing and plan limits
- Multiple workspaces with switcher
- Managed infrastructure (Fly.io API/worker, Neon Postgres, Cloudflare Workers)

## CE-only behaviour

- Single workspace per deployment
- Embedded Postgres and Redis in Compose
- `/setup` bootstrap when no users exist
- No billing surfaces

## Choosing an edition

Use **Cloud** for fast onboarding without operating infrastructure. Use **CE** when you need full control, air-gapped-adjacent deployments, or no subscription billing.
