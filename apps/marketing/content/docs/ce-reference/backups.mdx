---
title: Backups
description: Backup and restore guidance for PipeWatch CE Postgres data and configuration secrets.
---

## What to back up

- **Postgres volume** — all workspace, integration, and pipeline history data
- **Environment secrets** — `.env` or your secret manager export (never commit secrets to git)
- **GitHub App key** — store the PEM securely; loss requires generating a new key in GitHub

## Postgres backup

Take consistent snapshots while writes are quiesced or use `pg_dump` against the Compose Postgres service:

```bash
docker compose exec postgres pg_dump -U pipewatch pipewatch > pipewatch-backup.sql
```

Automate nightly dumps to off-host storage for production CE instances.

## Restore

Restore into a fresh Postgres volume before starting the API:

```bash
cat pipewatch-backup.sql | docker compose exec -T postgres psql -U pipewatch pipewatch
```

Verify migration version and run smoke tests against the dashboard before re-enabling webhooks.

## Redis

Redis holds ephemeral queue state. Rebuilding the queue after restore is acceptable; persistent Redis snapshots are optional for CE MVP workloads.
