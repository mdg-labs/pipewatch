title: Cloud quickstart
description: Create a PipeWatch Cloud workspace, connect GitHub, and see your first workflow runs in minutes.
---

PipeWatch Cloud is the managed edition at **cloud.pipewatch.app**. No Docker, database, or webhook tunnel setup required — PipeWatch hosts the API, worker, and dashboard for you.

## Prerequisites

- A GitHub account with access to the organizations or repositories you want to monitor
- Permission to install GitHub Apps on those accounts (org admin or personal account owner)

## 1. Create your account

Visit **[cloud.pipewatch.app](https://cloud.pipewatch.app)** and sign in with **GitHub OAuth**.

PipeWatch creates your first workspace automatically and assigns you the **owner** role. Existing users land on the workspace dashboard; first-time users enter the onboarding wizard.

## 2. Create a workspace

The wizard opens at `/onboarding?step=1`:

- Enter a **workspace name** (slug is generated automatically, editable).
- Choose a plan — **Free** is selected by default; upgrade prompts appear for paid tiers.
- Click **Create workspace** to continue.

You can create additional workspaces later from the workspace switcher (Cloud only).

## 3. Install the GitHub App

Step 2 explains the permissions PipeWatch requests and links to the official install URL:

```
https://github.com/apps/pipewatch/installations/new
```

Click **Install GitHub App**, choose the organization or personal account, and grant repository access. You can start with selected repositories and expand later in GitHub settings.

GitHub redirects back to PipeWatch with your `installation_id`. PipeWatch exchanges an installation token, creates an integration record, and discovers available repositories.

## 4. Select repositories

Step 3 lists repositories from the installation:

- Toggle repositories you want PipeWatch to track.
- Set ingestion mode per repo — **webhook** (default, real-time) or **polling** (fallback).
- Confirm to enqueue backfill jobs for historical runs (default 30 days on Free).

The dashboard populates as backfill completes. In-progress and new runs update live via SSE — no page refresh needed.

## 5. Explore the dashboard

After onboarding you land on the multi-repo dashboard:

- **Health bar** — repos currently green, failing, or running.
- **Repo cards** — last run status, failure rate, workflow count.
- **Run list** — filter by branch, workflow, status, or trigger; click through to job and step detail.
- **View on GitHub** — every run links to GitHub Actions for full logs (PipeWatch does not store raw logs).

## Settings and team access

| Area | Path | Notes |
|---|---|---|
| Integrations | Settings → Integrations | Manage GitHub App connection, re-sync repos |
| Members | Settings → Members | Invite teammates; roles: owner, admin, member |
| API keys | Settings → API keys | Workspace-scoped keys with `pw_` prefix |
| Billing | Settings → Billing | Plans, usage, Stripe portal (Cloud only) |

Members have read-only access; settings mutations require **admin** or **owner**.

## API access

Cloud exposes the same REST API as CE. Browse the interactive **[REST API reference](https://api.pipewatch.app/api/docs)** (Scalar) for endpoints, schemas, and try-it-out requests.

Authenticate with a JWT (browser session) or a workspace API key in the `Authorization` header.

## Self-hosting instead?

PipeWatch CE runs the same codebase via Docker Compose with all features free. See the [CE quickstart](/docs/getting-started/ce-quickstart) and [GitHub App setup](/docs/github-app-setup/creating-the-app) guides if you need full control or air-gapped deployment.
